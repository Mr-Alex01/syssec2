### Задание 1

`1. Активный master + пассивный slave`  
- master. принимает все записи (INSERT, UPDATE, DELETE)  
- slave. копирует данные, используется как резерв

`Преимущества`  
`1) Отказоустойчивость`  
Если master падает, slave можно быстро поднять как новый master и система не умирает полностью  
`2) Простота реализации`  
Простая логика, легко настроить (например, в mysql или postgresql)  
Часто используется в небольших проектах и как первый шаг масштабирования

`3) Резервное копирование без нагрузки на master`  
Можно делать бэкапы со slave и не тормозить основную систему

`2. Master + несколько slave`  
Это полная масштабируемая архитектура

`Преимущества`  
`1) Масштабирование чтения`  
- master. запись  
- много slave. чтение  
Если распределить нагрузку:  
- SELECT идут на разные реплики  
- система выдерживает гораздо больше пользователей

`2) Балансировка нагрузки`  
Через load balancer и shard, которая даёт равномерное распределение запросов и отсутствие перегрузки одного узла

`3) Высокая доступность`  
Если один slave падает - остальные продолжают работать  
Если падает master - один из slave становится новым master

`4) Геораспределение`  
slave можно разместить ближе к пользователям, благодаря чему уменьшается задержка

`5) Горизонтальное масштабирование`  
Если нагрузка растёт - достаточно добавить ещё slave

### Задание 2

`Принципы вертикального шардинга:`  
- разделение по бизнес-логике  
- независимость сервисов  
- снижение нагрузки  

`Принципы горизонтального шардинга:`  
- равномерное распределение данных  
- масштабируемость  
- уменьшение объёма данных на узел  

`Из минусов:`  
- сложность JOIN между шардами  
- усложнение логики приложения  
- необходимость балансировки нагрузки  

`Система использует комбинированный подход:`  
вертикальное разделение по сервисам (users, books, shops)  
и горизонтальное внутри каждой БД для масштабирования.  
Для отказоустойчивости применяется master-slave репликация,  
где master обрабатывает запись, а slave — чтение.   
`При отказоустойчивости базового master, slave становится новым master.`
